Minimalist Websocket Demo

This is an intentionally minimalist demo of websockets in action. This is the client portion, running in a web page. Show the "java console" for output information and show page source to see the javascript.

This web page establishes a connection to a websocket server running at the specified URL. This creates a TCP socket (with special websocket API sauce on top). The server side is configured to act as a hub--it acts as a message repeater, so as this page sends messages to the server, it will be repeated out to all other clients connected to the server. Likewise, this web page receives messages sent by other web pages.

Typically you would use JSON format messages. The heartbeat() function periodically sends a JSON message to the server. The message is extremely simple, in the form "{id: 6205}", or whatever randomly selected page ID this javascript generates. This web page will receive messages sent by other web pages, and process them in the onmessage() function. It assumes the messages it receives are in JSON format; it decodes the message and prints it on the console. NOTE: the javascript here is configured to contact a server running on the same host. If the server is running on a different host, you should change the SERVER_WEBSOCKET_URL variable.

For a more sophisticated example, see map.html from this same server. The map example uses google maps to display the location of icons on the map. The map expects to receive entity messages in a specific format: "{id:1234, latitude:36.6, longitude:-131.36}". You can modify the code in heartbeat() on this page to match that format.
